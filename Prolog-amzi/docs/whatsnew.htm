What's New

New Features

The bulk of the work done for Amzi! 5.0 has been in reworking the underlying structure of the system to better support our source code customers and to make porting and support on multiple platforms easier. As such, Amzi! 5.0 provides a better base for ongoing development that spans multiple platforms.

With Amzi! 5.0 we distinguish between the 'core' Amzi! + Logic Server system and the various Logic Server extensions offered with the product. The core is included for all platforms.

The extensions, as they always have been, are all open source, and are supported to various degrees on the different platforms. We welcome any and all contributions to the extensions portion of the system.

The core system includes:

  • The Prolog runtime engine, with full Unicode support, distributed as a dyanmic/shared library (.dll or .so). It is a Prolog virtual machine that supports, like Java, binary runtime portability across platforms.
  • The console development tools, including the listener/debugger (alis), the compiler (acmp), the Prolog linker (alnk), and the renameable runtime (arun). Renamed versions of the runtime can be used to create console applications.
  • The Logic Server API for integrating Prolog with C/C++. This enables both embedding of Prolog in C/C++ and the ability to develop extended predicates (LSX libraries) for Prolog in C/C++.
  • HTML documentation.

Extensions are built on the core and include:

  • Graphical development environments. (e.g. the Windows IDE)
  • Interfaces for various host languages to call the Logic Server API. (ex. Java, VB and Delphi support)
  • Extended predicate libraries. (e.g. ODBC, sockets)
  • Samples.
  • Applications (e.g. WebLS, SupportKT)

Changes

Initialization files are now called configuration files and have .cfg (instead of .ini) extension.
